# 2026-03-17

## 多账号配对 Phase 1（设置页内管理）
- 决策：保持现有 Settings 设计语言，不引入新视觉体系；账号管理复用 Connection 卡片区域、AppFont、状态 badge、按钮样式与 haptic 反馈。
- 实现：`CodexService` 增加多账号 profile 列表、active 账号、切换/重命名/删除、导入导出预留接口；配对保存从“覆盖单账号”改为“账号化 upsert + 切换 active”。
- 数据隔离：消息历史、AI change sets、runtime defaults、本地 archived/deleted 集合全部按账号 namespaced 分桶。
- 连接行为：断开连接不再清空已配对账号；仅显式删除账号时清对应 pairing。
- 诊断：账号记录 `lastConnectedAt` 与 `lastErrorMessage`，设置页按现有文案层级展示。

## 测试与修复
- 新增：`CodexRelayAccountManagementTests` 覆盖增删改切、切换后断网、删除后立即扫码新增。
- 修复：`stoppedTurnIDs` 计算改为“消息扫描 + turn->thread 映射”双来源，保证 interrupted completion 在无消息行时也能正确进入 stopped 缓存。
- 修复：将 `CodexRelayAccountManagementTests.swift` 与 `SettingsRelaySourceProbeStateTests.swift` 加入 `CodexMobileTests` target，避免出现 `Executed 0 tests`。
- 修复：`CodexRelayAccountManagementTests` 增加 `SecureStore` key 清理，消除跨用例污染。
