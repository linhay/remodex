# 设置页支持双源切换（LAN / Public Relay）

## 背景
- 当前已支持双源候选（LAN relay + Public relay）并在连接阶段自动排序。
- 现状只有自动策略，用户无法在设置页显式切换优先源。

## 目标
- 在设置页 Connection 区块新增“重试按钮 + 多源列表”。
- 列表点击可切换优先源（持久化）。
- 每个源行显示可达性与延时（毫秒）。

## 非目标
- 不修改 QR payload 协议。
- 不新增后端接口。
- 不改变现有断线自动重连机制与错误提示文案。

## BDD 场景
1. 当用户点击 `Retry` 时：
   - 立即触发一次重连尝试（已连接时先断开再重连）。
2. 当用户在列表点击任一源时：
   - 被选中源应成为连接候选第一优先级。
3. 当用户切回 `Auto` 时：
   - 恢复既有优先策略（LAN/Public 偏好逻辑）。
4. 重新启动 App 后：
   - 设置项应保持为上次选择值。
5. 进入设置页或点击 `Retry` 后：
   - 列表中的每个源探测 `/health`，分别展示：
   - `可达 · <xx>ms` / `不可达` / `探测中...` / `未配置`。

## 验收标准
- 设置页可见 `Retry` 按钮和多源列表。
- 连接候选 URL 排序与用户选中源一致（选中源优先）。
- 列表行展示可达性和毫秒延时，不显示速度档位文案。
- 单元测试覆盖策略排序、选中源优先、探测结果解析和持久化恢复。
- 对于 relay close code `4004`（auth mismatch）：不清空 saved pairing，不强制回扫码页，允许用户修复配置后点 `Retry` 恢复。
- Connection 区块提供恢复策略提示文案：`4004` 可重试恢复，session close/replace 仍需重新扫码。
- Connection 顶部来源文案优先显示“当前连接来源”；列表中单独区分 `Current`（当前连接）与 `Preferred`（下次优先源），避免语义混淆。
- 切换 `Preferred` 源后立即触发重连（仅在值实际变更时触发）。
- 若检测到“蜂窝网络 + 本地 relay 可达/当前在用”，展示本地直连解释提示（热点/USB/系统共享等）以减少误判。
- `Auto` 模式按探测延时优先选择可达源，并在前台按固定周期执行低延时重选切换。
- 自动重选切换成功后，在 Settings `Connection` 区块展示最近一次切换记录（来源 host、目标 host、目标延时、发生时间）。
