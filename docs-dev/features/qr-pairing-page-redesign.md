# 配对码页面视觉重构（与项目风格对齐）

## 背景
- 现有扫码页功能可用，但视觉层级和交互质感偏粗糙，与 Settings/Home 的卡片化风格不一致。

## 目标
- 在不改配对流程能力的前提下，重构 `QRScannerView` 的 UI：
  - 扫码主区域卡片化；
  - 顶部信息层级更清晰；
  - 手动输入入口更直观；
  - 权限缺失状态与主流程风格统一。

## 非目标
- 不变更 QR payload 协议。
- 不改连接业务逻辑（仅 UI/交互呈现优化）。

## BDD 场景
1. 相机权限已授权时：
   - 显示卡片化扫码区域、引导文案、手动输入与粘贴入口。
2. 相机权限未授权时：
   - 显示权限引导卡，并可直接跳转系统设置或改用手动输入。
3. 手动输入配对码：
   - 可粘贴、可提交；非法内容给出错误提示。
4. 剪贴板粘贴入口：
   - 剪贴板内容合法时可直接触发连接；非法时展示错误提示。

## 验收标准
- 页面视觉语言与现有卡片风格一致。
- 扫码、权限、手动输入三条路径功能保持可用。
- 现有 camera session policy 与 view model 单测不回归。
