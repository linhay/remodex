
## 2026-03-16
- 修复扫码页潜在内存压力：新增 QRScannerCameraSessionPolicy，仅在前台且未显示手动输入时运行 AVCaptureSession；弹窗/后台即停，回前台再启。
- 新增单测 QRScannerCameraSessionPolicyTests，覆盖 active/manual-entry/background 三类门禁，防止回归。
- 设置页支持双源切换（LAN/Public）：
  - 新增 `CodexRelaySourcePreference`（`auto`/`lanFirst`/`publicFirst`）并持久化到 `UserDefaults`。
  - `SettingsView` 的 Connection 区块改为：顶部 `Retry` 按钮 + 底部 `LAN/Public` 两张可点击卡片切换源。
  - `ContentViewModel` 的 relay 候选排序改为读取该偏好：
    - `lanFirst`：LAN 优先；
    - `publicFirst`：公网优先；
    - `auto`：保持原“探活 LAN，否则公网优先”策略。
  - 为排序策略新增单测覆盖三种偏好（当前放在 `CodexSkillsListDecodeTests`）。
  - 新增偏好持久化测试：设置后写入 defaults，重启后可恢复。
  - 双源卡片新增探测信息：显示 `可达性 + 延时(ms)`。
  - 探测触发时机：进入设置页与点击 `Retry` 后刷新；不做周期轮询。
  - `ContentViewModel` 补充探测结果模型（含 `latencyMs`）与 LAN/Public URL 映射辅助方法。
- 源数量增长后，设置区改为“多源列表”：
  - 列表每行展示 source URL、可达性、延时(ms)；
  - 点击某行设为优先源（持久化 `selectedRelayBaseURL`）；
  - `Auto` 可清除优先源，回退到原策略排序。
- 连接恢复策略调整：
  - relay close code `4004`（认证不匹配）不再清空 saved pairing；
  - 保留重连壳层，用户修复配置后可直接 `Retry`，避免被强制弹回 QR。
- 设置页 Connection 区块新增恢复策略提示文案，并补对应格式化单测，防止后续回归误导。
- 为避免 source 列表与顶部 Domain/来源文案打架：
  - 顶部改为优先显示当前真实连接来源；
  - 列表内区分 `Current` 与 `Preferred` 两种状态。
- 切换 `Preferred` 源后立即重连：
  - 仅在 `Preferred` 值发生真实变化时触发重连，避免重复点击导致无意义重连抖动。
- 增加“5G 但 local 源可达”的解释提示：
  - 监听当前是否有蜂窝接口；
  - 在“蜂窝 + 本地源可达/在用”时提示可能存在热点/USB/系统共享直连链路。
- 配对页 UI 重构：

  - 手动输入 sheet 增强输入框视觉层级与操作布局。
